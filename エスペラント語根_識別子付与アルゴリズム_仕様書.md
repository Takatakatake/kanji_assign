### （2025年9月の到達点・Google Apps Script 版 ／ **2026-06-16 PEJVO漢字化プロジェクトで全確定 → §0.5 が正典**）

## 0. この文書の位置づけ

漢字化エスペラントでは、語根への漢字・熟語の割当方針上、同じ漢字・熟語を複数の語根に再利用することを認めている（漢字種類数の最小化）。そのため、変換／復元アプリで「語根 ↔ 表記」を一意に復号できるよう、同じ漢字・熟語に割り当たった複数の語根を区別する **識別子**（上付き文字として表示する想定）を、後処理でまとめて付与する。

本書は、その識別子を機械的・一意に付与するために 2025 年 9 月に Google Apps Script 上で詰めたアルゴリズムをまとめたもの。

当時の状況を正確に書くと、**原理と全体の流れは確定**していたが、最後の枝葉（重複が起きたときの処理、子音と母音のどちらを採るかの一部の例）でコードの版が並走していた。本書では確定済みの部分を「確定仕様」、未収束の部分を「要確定」として明確に分ける。

### 0.5.1 適用対象と前提の具体化
- **A列 = エスペラント語根**(h-system表記。`ĉ`=`c^`/`ŝ`=`s^`/`ŭ`=`u^` 等。基礎語彙はUnicode表記のこともあるが、1文字単位で処理すれば同一)。**B列 = 割当簡体字/熟語**。
- **C列(優先順位) = `band_rank × 1000 + P`**(小さいほど上位)。
  - `band_rank`: 機能語・接辞・基礎語彙(basic/suf/pref/prep/correl/num/func)=**0**、PEJVO/学術/元素/暦/宗教(pejvo/sci/elem/cal/rel)=**1**、PIV/固有(piv/proper)=**2**。
  - `P` = §8の優先度指標(P = −V, V = a·log(F+1) + b·E − c·D, a=1.0/b=0.3/c=0.1, F=見出し数)。
  - 帰結: **層(band)が第一優先**、同層内はP(≈頻度降順)。差1.0以内は §3 の短語根優先が効く。
- **出力(識別子)は上付き文字で表示**。対応28字: `ᴬ ᴮ ᶜ ᶜ̂ ᴰ ᴱ ᶠ ᴳ ᴳ̂ ᴴ ᴴ̂ ᴵ ᴶ ᴶ̂ ᴷ ᴸ ᴹ ᴺ ᴼ ᴾ ᴿ ˢ ˢ̂ ᵀ ᵁ ᵁ̆ ⱽ ᶻ`。特殊字 `ĉ`(=`c^`)→`ᶜ̂`(基底superscript+結合曲折)、`ŭ`→`ᵁ̆`(+結合ブレーヴェ)等。

### 0.5.2 §6 の確定(要確定 → 確定)
- **§6(a) 衝突フォールバック = 別文字優先(数字を使わない)**。同一識別子が衝突したら、**子音列→全文字列の順で「最初の未使用文字」**を頭文字に付す。単一文字が尽きた時のみ数字(実データでは数字 0 件)。
  - 例: `bio` は `bi` が衝突 → 別文字 `bo`(=`生ᴮᴼ`)。`simbrank`→`ˢᴮ`、`acipenser`→`ᴬᴾ`。
- **§6(b) 差別化方式 = §5「位置ごとの新規文字」方式に一本化**(「基本形との最初の相違」方式は**不採用**)。
  - 帰結: `studado`(基本形`studi`)= **`sd`**(子音列の位置3で新規の `d`)。希望だった `sa` は採らない。`kakimono→km`・`strando→sn` を保つため §5 を正とする。
  - (注: §6(b)本文 L149「コードは子音で `st` を出した」は誤記。§5位置照合の実出力は `sd`。)
- **§6(c) 頭文字のみの範囲 = 「群内で頭文字が一意」 または 「同頭文字グループの初出メンバ(=その頭の代表)」なら頭文字のみ**。同頭文字に先行者がいれば必ず差別化文字を付ける。
  - 例: 一意=`faksimil→复ᶠ`。初出代表=`ripet→复ᴿ`(後続の同頭 r は差別化)。

### 0.5.3 位置照合のスコープ = per-head(頭文字グループ内)【重要】
- 差別化の位置照合(既出文字)は **同じ頭文字の語根とだけ** 行う(原典GASの `headGroups` 方式)。
- 理由: 識別子は同頭文字の語根とだけ衝突しうるので、別頭文字の語根を照合に混ぜると、衝突しえない相手に邪魔されて差別化字が劣化する。→ **group-wide(B群全体で照合)は不採用**。
- §5本文の「同グループ」は、最終版GASで per-head に精緻化されたものと解する。**付録Aコード(group-wide・§4 head-only分岐なし)は素朴な参考実装であり正典でない**(`uz→uz`・`sho→sh` 等、§5確定例と乖離する)。

### 0.5.4 §3 同点処理の厳密適用(確定)
- 優先度差 1.0 以内なら **短い語根を上位(基本形・短い識別子に寄せる)**。**頻度が高くても長い語根は短idを失う**ことを許容。
  - 例: `minut`(F7,5字)が `molekul`(F12,7字)より上位 → `minut=分ᴹ` / `molekul=分ᴹᴸ`。

### 0.5.5 確定した「際どいケース」(2026-06-16 ユーザー承認)
1. **重ね字の識別子は許容**: 位置照合で頭文字と同じ子音が新規になる場合、`r+r` 等の重ね字でも可。例 `reprograf=复ᴿᴿ`。
2. **近似同形の語根は区別記号差のみでも許容**: `anĉov=鱼ᴬᶜ̂` と `anchov=鱼ᴬᶜ`(h-system `ĉ` と `ch` の別語根)は `ᶜ̂` vs `ᶜ` の差で区別=視覚的に紛らわしいが現状許容。
3. **band優先の副作用は許容**: 低頻度の基礎語彙が高頻度の学術語より代表になりうる。例 `biologi(F0,basic)=生ᴮ` が `bio(F44,sci)=生ᴮᴼ` より上位。

## 1. 入出力（スプレッドシート前提）

処理は1枚のシートに対して行う。列の役割は以下のとおり。

- A列：語根（ラテン文字の読み。例 `kaku`, `strato`, `belo`）
- B列：割り当てた漢字・熟語（**グルーピングのキー**）
- C列：優先順位（数値。**小さいほど上位**）
- F列：**出力＝識別子**
- G列：処理マーク（2行以上の重複グループに「グルーピング済み」を付与）

---

## 2. 文字種の定義（確定）

識別子は語根の綴りから作るので、まず文字の分類を固定する。

- 有効文字：`a`–`z` および エスペラント特殊字 `ĉ ĝ ĥ ĵ ŝ ŭ`
- 母音：`a e i o u`（この5つのみ）
- 子音：有効文字かつ母音でないもの
  - → `ŭ` は **子音** 扱い（通常の `u` は母音）
  - → `ĉ ĝ ĥ ĵ ŝ` も子音
- 頭文字（head）：語根の最初の有効文字
- 大文字・小文字は区別しない（小文字化して処理）
- サロゲートペア対応（1文字単位で扱う）

「頭文字より後ろ」を対象に、次の2つの列を作る（後の差別化で使う）。

- 子音列：頭文字より後ろの **子音だけ** を出現順に並べた列
  - 例 `kakimono` → 頭文字 `k`、子音列 `[k, m, n]`
- 全文字列：頭文字より後ろの **有効文字（子音＋母音）** を出現順に並べた列
  - 例 `kakimono` → 全文字列 `[a, k, i, m, o, n, o]`

---

## 3. 全体の流れ（確定）

1. **B列でグループ化**する。空のB列はグループ化対象外（識別子を付けない）。
2. 行数が2以上のグループに、G列「グルーピング済み」を付ける。
3. 行数が2以上の各グループを **並び替える**。
   - 第1キー：C列の優先順位（昇順。小さいほど上位）
   - 第2キー（同点処理）：**優先順位の差が 1.0 以内** なら、A列（語根）の **文字数が短い方** を上にする
4. 並び替え後、グループ内を **上から順に** 処理し、F列に識別子を書き込む（→ 4節）。
5. 全行を一括で書き戻す。

> 補足：手順3の同点処理は「優先順位がほぼ同じなら短い語根を基本形に寄せる」という意図。差が 1.0 を超えるときは純粋に優先順位で並ぶ。

---

## 4. 識別子付与の確定原則

並び替え後のグループ内を上から処理しながら、次の原則で識別子を決める。

1. **最上位（1行目）の語根は識別子なし**（＝基本形。F列は空欄）。
2. **頭文字は識別子から決して省略しない**。識別子は必ず頭文字から始まる。
3. 区別の基準は **「頭文字＋子音」を徹底的に基準** とする。**子音だけで区別できない場合に限り、母音も基準に含める**。経験上、頭文字＋子音までで99%以上が区別できる。
4. 同じグループ内で **頭文字が他のどの語根とも異なる** 語根は、**頭文字のみ** を識別子とする。
5. 重複が生じたときは、数字を付ける前に、まず別の識別文字で一意化を試みる（→ 6節 b）。

---

## 5. 差別化文字の選び方（核となる手続き）

頭文字が他の語根と重なる場合、頭文字に「差別化文字」を1文字付ける。選び方は **「位置ごとの既出文字」との照合** で行う ＝ 先に処理した同グループの語根が、その位置に出していない文字を、先頭から探す。

**二段階**で探す。

- **第1段階（子音）**：子音列を先頭の位置から見ていき、「その位置で初めて現れる子音」を探す。見つかればそれを頭文字に付す。
  - 例 `kakimono` → 子音列 `[k, m, n]`、1番目 `k` は既出 → 2番目 `m` が新規 → **`km`**
  - 例 `strando`（基本形 `strato`）→ 子音列 `[t, r, n, d]`、`t`・`r` は既出 → `n` が新規 → **`sn`**
- **第2段階（母音・補完）**：子音だけでは新規が見つからない場合、母音について同じ照合を行い、最初の新規母音を頭文字に付す。
  - 例 `kaki`（基本形 `kaku`）→ 子音列 `[k]`（`k` は既出で打ち止め）→ 母音で差別化 → **`ki`**
  - 例 `strio` → 子音 `[t, r]` ともに既出 → 母音 `i` → **`si`**
- **どちらでも区別できない場合**（読みが実質一致）：頭文字のみ。なお最終手段として数字添字（`2`, `3` …）。

## 6. 要確定（2025年9月時点で版が分かれていた点）

> **【2026-06-16 すべて確定済 → §0.5 を参照】** 本節 (a)(b)(c) は当時の未収束点だが、現在は §0.5.2 で確定: **(a)=別文字優先 / (b)=§5位置照合に一本化(`studado→sd`) / (c)=「一意 or 同頭初出」で頭文字のみ**。位置照合スコープは **per-head**(§0.5.3)。以下は経緯の記録として原文を残す。

ここは当時、複数の挙動が並走しており、**実データで最終確認が必要**な部分。ごまかさずそのまま残す。

### (a) 重複時の処理 ── 数字添字か、別文字か

基本形 `bela` のグループで `belo` と `bola` が両方とも自然には `bo` を欲しがる、という衝突が起きる。

- コード上の素朴な挙動：`belo` → `bo`、`bola` → `bo` が重複 → 数字を付けて **`bo2`**
- あなたの希望：`belo` → `bo`、`bola` → **`ba`**（数字ではなく別の文字で区別）

当時の議論は「**数字より別文字を優先**（`bo2` ではなく `ba`）」に傾いていた。ただし「`bo` が衝突したとき、次にどの文字を採れば `ba` になるか」を、5節の位置照合方式だけからは一意に導けない（`bola` の `a` は基本形 `bela` の同位置の `a` と同じため、位置照合では『新規』にならない）。→ 衝突時のフォールバック規則を1つ決め打ちする必要がある。

### (b) 子音より母音が選ばれるべき例

5節は「子音優先」だが、次の例ではあなたの希望出力が母音を採っている。

- `studado`（基本形 `studi`）：コードは子音で `st` を出した → あなたの修正で **`sa`**
  - `sa` の `a` は、`studi` と `studado` を綴りの先頭から比べて **最初に食い違う位置の文字**（`studi` の `i` に対して `studado` は `a`）。
- `akvero`（基本形 `akvo`）：位置照合の子音方式だと `r` を採りかねない → あなたの希望は **`ae`**
  - `ae` の `e` も、`akvo` と `akvero` を先頭から比べて最初に食い違う位置の文字（`akvo` の `o` に対して `akvero` は `e`）。

→ つまり、`studado`・`akvero` は **「基本形との最初の相違位置の文字を採る」方式** の方が希望に合う。一方で `kakimono`（→ `km`）や `strando`（→ `sn`）は **「位置ごとの新規子音を採る」方式（5節）** でないと正しくならない。**この2方式は例によって食い違う。**

### (c) 頭文字のみで足りる範囲

基本形と頭文字が同じで、かつ他に同じ頭文字の語根がいない場合の扱い。

- `lerni`（基本形, 空）／ `lernado`：あなたの例では `lernado` → **`l`**（頭文字のみ。基本形が空なので `l` だけで区別できる）
- 一方 `bela`（基本形, 空）グループは `belo`・`bola` と **同じ頭文字が複数** いるため、頭文字だけでは足りず文字を付ける

→ 「基本形＋同頭文字が自分1人だけなら頭文字のみ、複数いれば差別化文字を付ける」という分岐になっていた可能性が高いが、ここも明文化して固定したい。

### まとめ（収束のさせ方）

(a)(b)(c) はいずれも「5節の **位置ごとの新規文字** 方式」と「**基本形との最初の相違文字** 方式」のどちらを正とするか、および衝突フォールバックをどうするか、の問題に集約される。**どちらか一方に決め打ちして、20〜30語の実セットで回し、食い違った事例だけ個別調整する**のが確実。

  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const sheet = SHEET_NAME ? ss.getSheetByName(SHEET_NAME) : ss.getActiveSheet();
  if (!sheet) throw new Error('対象シートが見つかりません。');

  const lastRow = sheet.getLastRow();
  if (lastRow < DATA_START_ROW) {
    SpreadsheetApp.getActive().toast('処理対象のデータがありません。');
    return;
  }

  // ========== データ取得 ==========
  const lastCol = Math.max(COL_F, COL_G);
  const range = sheet.getRange(DATA_START_ROW, 1, lastRow - DATA_START_ROW + 1, lastCol);
  const values = range.getValues();

  // ========== B列でグルーピング ==========
  const groups = new Map(); // key: B(trim), val: [ rowIdx, ... ] 0-based
  for (let i = 0; i < values.length; i++) {
    const b = toStr(values[i][COL_B - 1]).trim();
    if (!b) continue;                 // 空のB列はグループ化しない
    if (!groups.has(b)) groups.set(b, []);
    groups.get(b).push(i);
  }

  // ========== 重複グループにG列マーク ==========
  for (const idxList of groups.values()) {
    if (idxList.length >= 2) {
      for (const rowIdx of idxList) values[rowIdx][COL_G - 1] = 'グルーピング済み';
    }
  }

  // ========== 各グループで識別子付与 ==========
  for (const idxList of groups.values()) {
    if (idxList.length < 2) continue;  // 単独行はスキップ

    // 各行の情報を事前計算
    let rows = idxList.map((rowIdx) => {
      const aValue = toStr(values[rowIdx][COL_A - 1]).trim();
      const chars = toChars(aValue.toLowerCase());
      const headInfo = findHead(chars);
      const afterHead = extractAfterHead(chars, headInfo.index);
      const cNum = parseFloat(values[rowIdx][COL_C - 1]);
      return {
        rowIdx: rowIdx,
        original: aValue,
        aLen: Array.from(aValue).length,
        cValue: Number.isNaN(cNum) ? Number.POSITIVE_INFINITY : cNum, // 未指定は最後尾
        head: headInfo.char || '',
        consonantsOnly: afterHead.consonants,
        allChars: afterHead.all
      };
    });

    // 並び替え：C列優先 → 差が閾値以内なら文字数（短い順）
    rows.sort((x, y) => {
      const dc = x.cValue - y.cValue;
      if (Math.abs(dc) > PRIORITY_THRESHOLD) return dc;
      const dl = x.aLen - y.aLen;
      if (dl !== 0) return dl;
      return dc;
    });

    // 位置ごとの既出記録（子音用・全文字用の2系統）
    const seenConsonantsAtOrder = new Map();
    const seenAllCharsAtOrder = new Map();
    const usedIdentifiers = new Set();

    for (let k = 0; k < rows.length; k++) {
      const cur = rows[k];
      let identifier = '';

      if (k === 0) {
        identifier = ''; // 最上位＝基本形
      } else {
        // 第1段階：子音で差別化
        const c = findFirstDivergent(cur.consonantsOnly, seenConsonantsAtOrder);
        if (c) {
          identifier = cur.head + c.char;
        } else {
          // 第2段階：母音も含めて差別化
          const a = findFirstDivergent(cur.allChars, seenAllCharsAtOrder);
          identifier = a ? (cur.head + a.char) : cur.head;
        }

        // 衝突処理（※6節(a)：本来は別文字優先にしたい。ここは素朴版）
        if (usedIdentifiers.has(identifier)) {
          let n = 2, cand = identifier + n;
          while (usedIdentifiers.has(cand)) { n++; cand = identifier + n; }
          identifier = cand;
        }
      }

      values[cur.rowIdx][COL_F - 1] = identifier;
      if (identifier !== '') usedIdentifiers.add(identifier);

      // この行の文字を既出記録に追加
      updateSeenChars(cur.consonantsOnly, seenConsonantsAtOrder);
      updateSeenChars(cur.allChars, seenAllCharsAtOrder);
    }
  }

  // ========== 結果を一括反映 ==========
  range.setValues(values);
  SpreadsheetApp.getActive().toast('識別子の付与が完了しました（子音優先・母音補完版）');
}

// ==================== 補助関数群 ====================

function toStr(value) {
  return (value === null || value === undefined) ? '' : String(value);
}

// サロゲートペア対応の文字配列化
function toChars(str) {
  return Array.from(str);
}

// 有効文字判定：英字またはエスペラント特殊字
function isValidLetter(char) {
  return /^[a-zĉĝĥĵŝŭ]$/i.test(char);
}

// 母音判定（標準5母音）
function isVowel(char) {
  return /^[aeiou]$/i.test(char);
}

// 子音判定（有効文字かつ母音でない → ŭ は子音）
function isConsonant(char) {
  return isValidLetter(char) && !isVowel(char);
}

// 最初の有効文字（頭文字）を検索
function findHead(chars) {
  for (let i = 0; i < chars.length; i++) {
    if (isValidLetter(chars[i])) return { index: i, char: chars[i].toLowerCase() };
  }
  return { index: -1, char: '' };
}

// 頭文字以降を、子音のみ／全文字（子音＋母音）の2配列で返す
function extractAfterHead(chars, headIndex) {
  const consonants = [];
  const all = [];
  const start = (headIndex >= 0) ? headIndex + 1 : 0;
  for (let i = start; i < chars.length; i++) {
    const ch = chars[i];
    if (!isValidLetter(ch)) continue;
    const lower = ch.toLowerCase();
    all.push(lower);
    if (isConsonant(ch)) consonants.push(lower);
  }
  return { consonants: consonants, all: all };
}

// 「その位置で初めて現れる文字」を先頭から探す
function findFirstDivergent(charArray, seenAtOrder) {
  for (let i = 0; i < charArray.length; i++) {
    const order = i + 1; // 1-based
    const char = charArray[i];
    const seenSet = seenAtOrder.get(order);
    if (!seenSet || !seenSet.has(char)) return { char: char, order: order };
  }
  return null; // 全位置で既出
}

// 文字配列を「位置ごとの既出記録」に追加
function updateSeenChars(charArray, seenAtOrder) {
  for (let i = 0; i < charArray.length; i++) {
    const order = i + 1;
    const char = charArray[i];
    if (!seenAtOrder.has(order)) seenAtOrder.set(order, new Set());
    seenAtOrder.get(order).add(char);
  }
}
