## 2. 確定方針（カテゴリ別・確定の経緯つき）

### 2.1 漢字選定の5原則（原典 R1〜R5、現在の確定形）
- **R1 字種最小化**: 使う漢字の種類を最小化し**使い回しを歓迎**。短い形（1字・既割当の共有字含む）が中国語で正しく意味するなら**必ず共有**せよ（球×6・帆×2 OK）。「既割当だから」を理由に字を足す/新字導入するのは**厳禁**。字を足すのは、短い形が中国語標準で『別物』を指し学習者が誤読する**真の false friend** の時だけ（その場合も既割当字の組合せで作り新字を避ける）。**経緯**: 長尾で検証パスが「既割当だから」と新字を導入するドリフトをユーザーが発見・是正（papaj 番木瓜→木瓜、kamomil 洋甘菊→甘菊、ĵib 前帆→帆、kopal 树脂→脂）。
- **R2 基礎漢字**: 一级3500字を強く優先（HSK基礎の機械照合可能化）。**現確定では割当字は一级のみ**（後述2.3）。
- **R3 画数・学習効率**: 省エネ（少画数）。ただし R4 が優先。
- **R4 意味透明性（最重要）**: 音訳・部分音訳は**絶対禁止**。日中双方で意味がなじむ字を選ぶ。日中乖離ルール（必須置換）: 好→良 / 房→室 / 写→书 / 吃→食 / 城→市 / 面→颜 / 図→画（「絵」義のみ。地図/図表の图は中国語標準で維持）。
- **R5 体系的一貫性**: 一貫性 > 自然さ。**英語対応語の中心義を多義語根の基準義とする**（国際的理解促進、v2監査で確定原則として復活）。同語源・類似音は共通核漢字で系統化（konkur=竞 / konkurs=竞技）。

採否比較順: ①意味透明性（日中双方）→②基礎漢字該当→③文字数→④画数。

### 2.2 字数ルール（最重要・最終確定）
**原則1字・最大2字**（省エネ最優先。汎用度の高い常用語根は必ず1字）。
- 1字がその語根の中心義を担えば1字維持（应=must / 界=world / 解=understand / 头=head / 问=ask / 末=last / 良=good / 余=remainder は全て1字で十分）。**thin な1字より意味が立つ1字**（animal は动でなく**兽**）。
- 多字（2-3字）は2類のみ: (a) その1字の主要義が語根の意味を含まない・無関係（使=使う→angel は**天使**、号=番号→trumpet は**喇叭**、动=動く→animal は**兽**）、(b) 稀な専門・動植物・器物で適切な1字が無い（目皮=eyelid、物理=physics、孔雀=peacock、山毛榉=beech）。
- **3字以上は真に不可分な極稀専門概念のみ**（2字化を最優先: 蛋白质→蛋白、手榴弹→手雷、黄道带→黄道、机车=lokomotiv、上院=senat、实验=laboratori、伞=paraŝut、细菌=mikrob 等）。維持例外: 月名 十一月/十二月、樹木 山毛榉/无花果/仙人掌/悬铃木/猫头鹰、鉱物 大理石/安息香、専門 反应堆/染色体/核糖体/安乐死/内分泌/鸡尾酒/肉豆蔻/龙舌兰/共济会/无政府/男高音/女高音 等。
- **経緯**: 一度「多字許容」に振れ、透明性監査が単字内容語2376件の55%（1304件）を過剰に2字化提案。ユーザーが「dev=应 kompren=解 mond=界 kap=头 demand=问 last=末 全部十分。最初の方針しっかり見てね。語自体の汎用性も考えろ」と強く是正し、**監査結果1304件は全破棄、原則1字へ引き締め**。その後3字熟語への難色（「蛋白质/反应堆/办公室/手榴弹 この程度で3文字はな」）で**最大2字**を確定。詳細メモ: `feedback_evocative_kanji.md`。

### 2.3 字種ルール（最終転換: 一级のみ）
**割当に使う漢字は一级3500字のみ**。一级で透明に中心義へ到達できなければ**二级で妥協せず未対応**にする（=ラテン保持）。
- **経緯**: 当初「二级まで許容」（_nivelo2.tsv に記録）→ユーザー「全て一級に収めたい」→一级化リダクションで二级298→188→さらにユーザー「**二級がどうしても不可避なら、もう未対応でいいよ。一級だけにして**」で二级全廃を確定。
- **「一級字に何が何でも収める」の正しい意味**=「漢字を割り当てる際に**使う字を必ず一级にせよ**」という字種ルールの強調。**全語根を必ず割り当てる/未対応を禁じる意味ではない**（未対応は今後もOK）。
- **生物種例外（拡張: 生物種に限らない）**: 一级に専用字がない具体物は、一级内の**最も近い上位概念字**で救済してよい。果実→果、樹木→木、草本→草、鳥→鸟、魚→鱼、虫→虫、海藻→藻、羊→羊、茶→茶、酒→酒。大型トカゲ（igvan/varan）→龙（evocative）、小型（scink/agam）→龙は大げさなので未対応。**元素・系統化学・解剖部位・病気は生物種でないので未対応維持**。救済時は救済語根自体が homonym だと第2義に化けるので**辞書実見出しでの裏取りが必須**（piran=ピラン化学≠ピラニア、arame=アラム人≠海藻 を誤救済→撤回した教訓）。
- 字種監査: `_audit_jizhong.ps1`（一级基準）で非一级を検出。現在**一级外0**（真の0達成）。

### 2.4 元素（カテゴリ字救済、2026-06-20確定）

### 2.5 識別子付与アルゴリズム（§9、正典確定）
同じ漢字を複数語根が共有してよい（**衝突歓迎**。最後に上付き識別子で区別）。正典は**原典GAS「最終完全版」**（`_gas_resent.js`、仕様書 `エスペラント語根_識別子付与アルゴリズム_仕様書.md` §0.5）。実装は `_gas_identifier.ps1`。

確定アルゴリズム:
1. 同じ漢字でグループ化（1語根のみなら識別子なし）。
2. ソート: **C = band_rank×1000 + P** 昇順。**優先度差1.0以内なら短い語根を上位（=基本形）**（§3厳密。minut=分ᴹ が molekul=分ᴹᴸ より代表）。
3. 最上位＝基本形（識別子なし）。
4. 非基本形の id 付与順 = band昇順→F降順→短語根→P。**頭文字は必ず保持**。頭文字が群内一意、または同頭文字グループの初出メンバなら**頭文字のみ（head-only, §4）**。
5. 差別化字 = **per-head 位置照合（§5）**: 同じ頭文字の先行語根とだけ位置ごとに照合し、その位置で初めて現れる文字を採る（**子音列優先→なければ全文字列で母音補完**）。基本形もseenに寄与する。例: komplik=复ᴷᴾ / kompleks=复ᴷˢ。
6. **衝突フォールバック = 数字でなく別文字（§6a）**（子音→全文字で最初の未使用）。**数字id 0件**。
7. 識別子は**全語根が出揃った後の最終全体パスで一括付与**。

**band_rank**: 機能語・接辞（suf/pref/prep/correl/num/func）+基本語彙CSV = **tier0（最上位）**、PEJVO/学術 = 1、PIV/固有名/元素/暦 = 2。これにより el=出（F449）/sen=无（F689）/ig=使（F2032）/ad=行/aĵ=物/ist=家 等の最基本機能語・接辞が必ず基本形になる。**band差≈1000が字数閾値1.0を圧倒**するので band が支配的。

### 2.7 P（優先順位指標）の数値化
P = −V、V = a·log(F+1) + b·E − c·D。**a=1.0 / b=0.3 / c=0.1**。F=見出し数（汎用性）、E=ラテン長÷総画数（省エネ度）、D=画数+字表ランク（習得難易）。自然対数 ln。C=band_rank×1000+P。層バンドは跨がない。`_p_work.csv` で計算。

### 2.8 homonym（同綴り異義）— 2型＋comb
学習者版は whole-spelling 衝突を既に単一分解へ統一済（##偽分解マーク）。segment級は `_homonym.tsv`（73行=ヘッダ＋sep31＋amb40＋comb1）で管理し最終注入で適用。
- **sep（31件・注入適用）**: 第2義が別見出し文字列→見出しキーで上書き。例: ĉar=因 vs ĉar/o=车、al=向 vs al/o=翼ᴬ、por=为 vs por/o=孔ᴾ、sur=上 vs sur/o=腿ˢ、el=出 vs el/o=酒ᴱ、mark=标 vs（貨幣）。元素 krom/o=金ᴷᴹ も sep。
- **amb（40件・不採用）**: 同一見出し文字列に2義併存（mark/o=标&币、karp/o=鲤&腕、spin/o=脊&旋、sinus/o=洞&弦）。**検出困難＋「伸ばせば一義（oz/it型）」が多いためユーザー裁定で注入では不採用**（主字共有に倒す、第2義は辞書文【貨】等が示す）。台帳には記録のみ残す。
- **comb（1件）**: 段位置一律の結合形。fon→声（-fon-=音、idx>0完全一致のみ。font/fond/ŝifon は別分節で不発火、背景 fon/o=底 は idx0 で不変）。
- **結合形（ギリシャ）同綴衝突 4語根**（多義のため comb でなく見出し列挙 sep で正当語保護）: **fil→爱ᶠ**（愛書家 bibli/o/fil。filo=息子 bo/du-on/adopt-fil は儿維持）、**lit→石ᴸ**（隕石 aer/o/lit。lito=寝台 du-lit は床維持、elektrolit=-lyteは除外で床維持）、**te→神ᵀ**（無神論 a/te/ism、神学 te/o/logi。buter/te=茶維持）、**top→境ᵀ**（生境 bio/top=habitat。top/o=檣楼の帆楼維持）。
- **oz/it型**（伸ばせば一義に解釈可）はメジャー義の漢字を**共有**してよい（分割しない、台帳に入れない）。
- **非一级3件**（bram=桅 / tik=痉 / prote=螈）は第2義が一级に無く主字維持（除外）。
- 偽sep（false sep）: 同一見出しに主義+第2義が同居する語（abak/o=算盘+冠板 等16件）は `$forceAmb` で sep→amb に再分類して主義復帰。
- 生成: `_build_homonym.ps1`（$existing+$new+$comb+$forceAmb+$forceUnt）→ `_homonym_disp.ps1`（第2義のdisp計算）。

### 2.9 privative / 連結母o / 文法語尾
- **privative an-/a- = 无ᴬ**: 語頭 a-/an- で**直後が実語根**の時のみ発火（a/morf/a⟦无ᴬ/形ᴹ/a⟧、an/estez/i⟦无ᴬ/感/i⟧）。直後が派生接尾辞（ar/iĝ/ec 等=`$sufSet`）だけなら不発火→an は接尾辞 员（an/ar/o⟦员ᴬ/群⟧=会、an/iĝ/i⟦员ᴬ/成⟧=入会）。**ガードを 2026-06-20 追加**。
- **endingRe は on/an/en を除外必須**: -on=分（中位分数 du/on/o⟦二/分/o⟧）、-an=员、en=内 を発火させる。**終端の対格 -on/-an**（語の最終分節 idx>0）は**ラテン保持**（猫/on）。区別は終端か否か。
- **連結母o = 保持（1:1構造、`$dropLinkO=$false`）**: akv/o/fal/o⟦水/o/落/o⟧、bel/ec/o/krem/o⟦美/性/o/酪/o⟧。**経緯**: 省略/保持で2度往復（アシスタント誤解釈）→ユーザー「接尾のoを省略してよいといつ言った？」「連結o=保持(1:1構造)、形態素境界を崩さない」で確定。省略は後処理に委ねる（`$dropLinkO=$true` で再有効化可、その場合 水落/o）。
- 文法語尾（-o/-a/-i/-e）は常にラテン保持。

### 2.10 分詞相字
能動 -ant=**在**（進行）/ -int=**过**（経験完了）/ -ont=**将**（将然）。受動 -at=**被** / -it=**受** / -ot=**待**。**ただし化学塩接尾辞 -at/-it は塩**（後述2.11）。区別は語釈ベース。

### 2.11 化学塩 -at/-it = 盐（透明化）
- -at→**盐ᴬ**、-it→**盐ᴵ**。酸根は通常disp（acet=醋/karbon=碳/sulf=硫/fosf=磷/nitr=氮/silik=硅ˢᴷ）。acet/at→醋/盐ᴬ。sal=盐（塩そのもの）と識別子で一意。
- **経緯**: ユーザー「化学塩 -at/-it = 盐 で透明化（酸根の漢字を活かしR4向上）。これでよろしく頼む」。当初は化合物=未対応にしていたのを透明化に変更。
- **判別子は行レベル語釈ベース `$chemSaltLine`**（`_inject_final.ps1`）: ①語釈に `Sal[oj]`（Salo de/aŭ、酸塩、酸盐）②見出しに独立語 acid/o ③任意分節が酸根（$chemAcid。tartr/sakar 追加済、Metalderiva/Metalkombina トリガ追加済）。**中位は bor 除外**（钻ドリル動詞と同綴 verm/bor/it=虫食い受動の誤爆防止）。
- 受動分詞は無影響（am/at⟦爱/被⟧維持、全コーパス1068件）。複雑有機/官能基/系統化学/ビタミン、化学アルコール -ol（過細分解17件）は未対応。

### 2.12 固有名 = 一律未対応
国名・地名・民族名・言語名・人名・神名は、**中国語名が透明（japan=日、中国）でも一律未対応（ラテン保持）**。漢字圏贔屓を避けるため。透明CJK名の国だけ漢字を与えるのは東アジア優遇=不公平。ただし翻訳可能な文化・概念語（samuraj=武士、浮世绘、挂轴）は意味訳可。宗教語も意味字あれば訳（budh=佛/di=神/preĝ=祷/anĝel=天使）、音訳のみは未対応（**Krist未対応**、islam/Koran未対応）。
- 注入の固有名ガードは**語単位（per-word）の先頭大文字**で発火。全語が大文字始（Japan/a Mar/o）なら⟦⟧無し、一部が小文字内容語なら部分注入（Amerik/a bizon/o⟦Amerik/a 野牛/o⟧）。
- **単一大文字+ハイフン語は除外**（T-/U-/X-/H- は型・略号で固有名でない）→ハイフン分解に到達させる（T-ĉel/a→T-胞/a、U-form/a→U-形/a、X-radi→X-射ᴿ、H-bomb→H-弹）。多文字大文字固有名（Baden/land、Kolomb/i/o）は保護維持。ガードは `_inject_final.ps1` L50: `(-cmatch '^[A-Z…]') -and (-notmatch '^[A-Z…]-')`。

### 2.13 ハイフン分解 / 結合形
- ハイフン `-` は `/` と同じ形態素境界として**必ず分解**（「-は/とほぼ一緒。割り当てが分かれてないとか絶対にありえない」）。whole-word ハイフン entry（ĉi-jar=今年 等）は master から全廃し、各形態素を漢字化してハイフン連結（此-年、菠萝-汁、alfa-粒ᴾ）。
- 近接小詞 **ĉi=此** を新規導入（相関詞接頭 ĉi-=全 とは別語）。ĉi-jar/e⟦此-年/e⟧、ĉi ĉio⟦此 全o⟧。
- 結合形（combining-form）の同綴衝突是正5系統: -fon-→声、科学nom（agronom/astronom）→学ᴺᴼ（名前複合 famili/nom は名のまま）、al/de接頭+log→诱ᴸ（bio/log等の科学は学家のまま）、term（thermo）→热（項=誤り。用語は別語根 termin=词）、化学塩-at→酸根+盐。

## 7. 教訓・落とし穴

**PowerShell の罠**:
- 変数名は**大文字小文字を区別しない**。`$BR`（ハッシュ表）を `$br`（ループ変数）が上書きする、`$ITEMS`/`$items` clobber、`$F`/`$f`、`$cc`/`$cC`、`$Skip`/`$skip` の衝突が頻発→必ず変数名を分離（lo_/up_ 接頭等）。`$ITEMS=($items|ConvertTo-Json)` は `.Count=1` の誤表示を招く（JSON自体は正）。
- **配列フラット化**: `@( @('fon',...) )` がパイプで文字ごとに分解されゴミ台帳行を生む→単項コンマ `@( ,@(...) )` で回避。
- `.ps1` BOM無し→PS5.1 が Shift-JIS 誤読し日本語パス/出力破損→BOM付き再エンコード。`& $path` で同一セッション実行。
- `Select-Object -First N` が上流スクリプトのパイプラインを早期停止させファイル書込前に中断。
- ハッシュテーブル case-insensitive で固有名 `Mar/o`（未対応）と内容語 `mar/o`（海）を同キー混同→Ordinal Dictionary + 割当優先で is区別。Budh/budh、Bibli/bibli も同種（Bibli=圣 削除し bibli=书 に一本化）。
- `$h:` のコロンをドライブ修飾と誤解→`${h}` で区切り。`-File` の子プロセスがUTF-8(BOM無)をANSI誤読。

**doc 陳腐化**: 方針書の §14「現状」スナップショット（件数 homonym/master/サイドカー/被覆/救済段階/日付）が取り残されやすい→**実ファイルを再計数して同期**。`§N` の節番号は行番号化ミスに注意（§73=行73）。識別子の具体例も最終アルゴリズム（§3厳密・base上書き）に未追従でズレやすい（molekul=分ᴹ→分ᴹᴸ 等）。

**塩/受動の判別**: -at=被（受動分詞）が塩接尾辞に化ける系を**単体監査（ハード制約）は見逃す**→敵対監査で捕捉。判別は**語根リスト限定でなく行レベル語釈ベースが堅牢**（接頭辞・有機酸・同綴動詞で語根限定は破綻）。

**固有名ガードの誤爆**: per-word 先頭大文字ガードがハイフン型接頭（T-/U-/X-/H-=型・略号で固有名でない）を誤爆しハイフン分解を阻む→単一大文字+ハイフンは除外、多文字大文字（固有名）は維持。

**検証パスは誤りを出す**（語義照合で人が捕捉せよ）: sopran→女高（=女子高!）、ameb→变形（=変形）、globin→球蛋白（=globulin）、poez=生（=詩の誤）、observatori→天台、obez→目鲔、impon→慑（方向逆）、karikatur→讽画、vokal→音（母音/子音区別不能）。検証パスは final が自分の reasoning と矛盾する型の誤りを出す（u̯at=瓦 を「瓦=音訳で不可」と言いつつ採用）。**最終注入前に全割当の語義照合QAパスを1回かける**（`_gen_semqa.ps1`/`_wf_semqa.js` 構築済・未実行）。

**プラットフォーム不調**: 高並列ワークフローで「subagent completed without calling StructuredOutput」が多発（一過性 blip、負荷で悪化）→**並列波は撤回し逐次実行**。失敗バッチは master 取込前なので無傷→同スクリプト再実行で復旧。

**救済の落とし穴**: 救済語根自体が homonym だと第2義に化ける（piran=ピラン化学≠ピラニア、arame=アラム人≠海藻）→辞書実見出しでの裏取り必須。

**設計上不可避**: 1語根=1字は中心義しか捉えず、多義語根（afer=事だが affair/business/訴訟…）は派生・複合で字とズレた読みが出るのは設計上不可避。
