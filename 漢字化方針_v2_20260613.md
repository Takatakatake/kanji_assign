## 1. 最終目標と設計思想

作るのは「自然な訳語辞典」ではなく、**規則から未知語の意味を予測できる、閉じた・体系的な表記コード**。対象は日本人・中国人**双方**+**非漢字圏学習者**。簡体字基本。個々の自然さより**透明性・一貫性・最小性**を優先する(これが全判断の上位制約)。

## 2. 用語定義

| 用語 | 定義 |
|---|---|
| 語根 | スラッシュ分解形態素のうち文法語尾以外。接辞も割当対象。 |
| 文法語尾 | -o/-a/-i/-e/-as 等。**漢字化せずラテン字のまま残す**(§3・§12)。 |
| 割当漢字 | 1語根=1漢字、または各字が意味に寄与する透明2字。 |
| 核漢字 | あるカテゴリを代表し複数語根に再利用する字(R1)。 |
| 未対応 | 意味訳不能でラテン語形のまま残す語根。 |
| 識別子 | 同字共有時に優先順位の低い語根へ付す上付き標識(§9)。 |
| 基本形 | 同字グループで最優先(band → F → 短語根。§8 の base 上書きあり)の語根。識別子なし。 |
| P | 優先順位指標。高いほど優先順位が低い(§8)。 |
| h-system | 辞書本体の `c^=ĉ` 等のキャレット表記(§10)。 |

## 3. 割当の対象と単位 ★

- 見出しの**スラッシュ分解を信頼**し、**各語根・接辞に漢字を1対1対応**。勝手な再分解・語尾剥がしはしない(綴り不変・偽分解回避)。【確】
- **ハイフン `-` も `/` と同じ形態素境界**として必ず分解する【確・2026-06-20】。`ĉi-jar/e⟦此-年/e⟧`(ĉi=此 + jar=年)・`ananas-suk/o⟦菠萝-汁/o⟧`・`advokat-konsil/ant/o⟦辩ᴬ-议/在/o⟧`・`alfa-partikl⟦alfa-粒ᴾ⟧` のように各形態素へ漢字を当て `-` で連結(見出しの `-` を保持)。**ハイフン語を1語まるごとに漢字化してはならない**(`ĉi-jar=今年` 型の whole-word 割当は禁止。master のハイフン root は全廃済)。`_inject_final.ps1` がセグメントを `/` だけでなく `-` でも分割。
- **文法語尾は漢字化せず、ラテン字のまま `⟦…⟧` 内に残す**。【確・2026-06-13 訂正】
- 接辞・相関詞・前置詞などの機能語も対象。【確】

```
見出し  abel/bred/ist/o
割当    蜂 / 养ᴮ / 家 / o    ← 語根→漢字、接辞 ist→家、語尾 o はそのまま(bred=养ᴮ は基本形 nutr=养 と同字共有→識別子 ᴮ)
出力    abel/bred/ist/o⟦蜂/养ᴮ/家/o⟧:養蜂家
```

## 4. 漢字選定の原則・手順

### 4.1 5大原則【確】

| | 原則 | 要点 |
|---|---|---|
| R1 | 漢字種類数の最小化 | 同字・偏旁・熟語を再利用。1字で立つなら1字。2字は各字が意味に寄与する時のみ(冗長2字化不可) |
| R2 | 基礎漢字 | HSK基礎・簡体字・**《通用规范汉字表》一级(3500)**内。専門難字は基礎字の組合せで代替 |
| R3 | 画数・学習効率 | 少画数優先。多画数難字1字より少画数の透明2字 |
| R4 | **意味の透明性(最重要)** | **音訳は部分音訳も含め絶対禁止**。各要素から全体意味が論理的に推測可能(mikroskop→微镜) |
| R5 | 体系的一貫性 | **一貫性 > 自然さ**(不自然でも統一適用。回避は既存語と衝突し誤読を招く時のみ)。接尾辞は完全統一。**同語源・類似音は共通核漢字で系統化**(konkur竞/konkurs竞ᴷˢ)。**多義語根は英語対応語の中心義を基準義**とする(国際的理解の促進)【確】 |

### 4.2 探索順(候補生成)【確・2026-06-16 改訂】
① HSK基礎1字(一级) → ② 一级字どうしの透明2字熟語 → ③ 専用字が無ければ**最近一级字の代用**(§4.6) → ④ 未対応。
※ 旧版の「③ 部首が意味分野を示す一级外字」は**二级全廃(§12)に伴い廃止**。候補は**常に一级のみ**。

### 4.3 採否比較順(候補選択)【確】
**① 意味透明性(日中双方) → ② 基礎漢字該当 → ③ 文字数 → ④ 画数。**

### 4.4 日中乖離の裁定(R2c)【確】
主要意味が日中で食い違う字(走・汤・手纸、写=写す・吃=どもる 等)は、**同等に易しい代替があれば回避**。代替が難字しかなければ**中国語側の意味を基準**に採用。
> 実適用例: 好→良 / 房→室 / 写→书 / 吃→食 / 城→市 / 面→颜 / 図→画。

### 4.5 絶対禁止【確】
音訳(部分音訳含む)/ 意味不透明な慣用熟語 / **一级外字・二级字**(=一级のみ確定 §12。例外なし)/ 冗長2字化 / 漢字圏固有名詞への既存表記流用。

### 4.6 最近一级字の代用【確・2026-06-16 / 拡張 2026-06-20】
専用の一级字が無い語根(特に生物種)は、**未対応にせず《一级にある最も近い上位概念字》で救済**してよい(漢字化優先度は最低でよい=優先度を上げる意味ではない)。
- **生物**: 魚→鱼 / 鳥→鸟 / 昆虫→虫(蝶・蜂・蛛も)/ 草本→草 / 木本→木 / 藻→藻 / 苔→苔 / 哺乳一般→兽(サル猴・ネコ猫・イヌ犬・ウマ马・イノシシ猪・シカ鹿・サイ犀・ネズミ鼠・コウモリ蝠)/ ヘビ蛇 / カエル等両生→蛙 / カメ龟 / カニ蟹 / エビ虾 / 貝贝 / 細菌・真菌→菌。
- **生物以外にも拡張可**(明確な上位一级字があるカテゴリ。例 urn つぼ→缸 / 大型トカゲ(イグアナ・オオトカゲ)→龙)。**要個別判断**(小型トカゲ等、代用字が過大なら未対応に倒す)。
- **元素【2026-06-20】**: 一级専用字を持つ元素は維持(铁/铜/银/铅/锡/锌/硫/磷/氯/碘/锰/钠/硅 等)。専用字が二级/表外の元素は **金属→金 / 希ガス・気体→气**(=最近一级字のカテゴリ適用)で救済。例 titan/o⟦金ᵀᵀ⟧・urani/o⟦金ᵁ⟧・kobalt/o⟦金ᴷᴮ⟧・neon/o⟦气ᴺ⟧・fluor/o⟦气ᶠ⟧。クロム/チタン等は同綴(prep krom=外・giant titan=巨)と homonym 分離(§6.5)。**半金属(bor/selen/telur/antimon/arsen)→矿(鉱物)・液体ハロゲン(brom)→卤(ハロゲン)**【2026-06-20確定】(金属でも気体でもない=別カテゴリの最近一级字。bor/o⟦矿ᴮ⟧・arsen/o⟦矿ᴬ⟧・brom/o⟦卤ᴮ⟧。bor は掘削 bor/i=钻 と homonym 分離)。hidrogen/oksigen は分解合成(hidrogen=水/生・oksigen=氧/生)で対象外。
- **化学塩・酸 -at/-it【2026-06-20 改】**: 酸根の漢字 + **盐**(塩)で透明化(`acet/at/o⟦醋/盐ᴬ⟧`酢酸塩・`sulf/at⟦硫/盐ᴬ⟧`・`nitr/at⟦氮/盐ᴬ⟧`・`sulf/it⟦硫/盐ᴵ⟧`亜硫酸塩。盐ᴬ=-ate/盐ᴵ=-ite)。**受動分詞-at/-it=被/受 は非化学語で維持**(`am/at/o⟦爱/被⟧`恋人・`arm/it/a⟦武/受⟧`武装した・`verm/bor/it/a⟦虫/钻/受⟧`虫食い)。**判別は行レベルの語釈ベース**(旧 `$chemAcid` 先頭分節限定では金属オキソ酸塩 `fer/plumb/stan/zink`・接頭辞付き `per-/meta-/bi-/hipo-/tio-`・有機酸塩 `tartr/ure/glutam/benzo` を取りこぼし -at が誤って受動分詞「被」に化けていた→是正)。**`$chemSaltLine` の発火条件**: ①語釈に `Sal[oj] `(Salo de/aŭ・Saloj de)・`酸塩`・`酸盐`・`Metalderiva`/`Metalkombina`(金属誘導体/化合物=塩。`sakar/at⟦蔗/盐ᴬ⟧`糖酸塩・`etanol/at⟦etanol/盐ᴬ⟧`アルコキシド) ②見出しに独立語 `acid/o`(酸形 `benzo/at/a acid/o⟦安息香/盐ᴬ/a 酸⟧`) ③任意分節が酸根(`$chemAcid`=acet/fosf/karbon/nitr/sulf/silik/molibden/klor/brom/jod/krom/bor/arsen/fluor/volfram/vanad/selen/telur/antimon/**tartr/sakar**。tartr/sakar は語釈に塩語が無い `吐酒石 vom/ig/a tartr/at/o⟦吐/使/a 酒石/盐ᴬ⟧` 等を radical で確実化。**中位は `bor` 除外**=钻ドリル動詞と同綴の `verm/bor/it`=虫食い受動 の誤爆防止。boron塩 `bor/at` は先頭分節+homonymで処理)。該当行の `-at`→盐ᴬ・`-it`→盐ᴵ。受動分詞は無影響(`am/at⟦爱/被⟧`・`verm/bor/it⟦虫/钻/受⟧`)。酸根字は homonym(`krom/titan→金`・`bor→矿`)/disp(`acet=醋`・`fer=铁`等)で。酸根が一级外で無い未対応語根は latin 保持(`cian/盐ᴬ`・`ftal/盐ᴬ`)。§6.5・`_inject_final.ps1`。
- **救済しない(未対応のまま正しい)**: 複雑有機化合物・官能基・系統化学(IUPAC名)・ビタミン(烯/醛/kalciferol 等)、固有名・地名・神話・宗派、間投詞(on!/e!)・字名/略号(l/d/k/beta)。

## 8. 優先順位指標 P ★

「高いほど優先順位が低い」指標。3要素を合成(ユーザー指定):

| 要素 | 定義 | 向き |
|---|---|---|
| **F 汎用性** | 辞書でその語根を含む見出し数 | 高→優先↑ |
| **E 省エネ度** | 語根のラテン文字数 ÷ 割当漢字の総画数 | 高(省エネ)→優先↑ |
| **D 習得難易** | 画数 + 字表ランク | 高(難)→優先↓ |

- 合成 `V = a·ln(F+1) + b·E − c·D`(自然対数)。**層(基本語彙→PEJVO→PIV)は別バンドで上書きし跨がせない**。**P = −V**(小=高優先=基本形・良い漢字を優先取得)。
- 層内の基本語彙は **Unified_Level** を第一キー(低いほど優先)。
  - ※ founding prompt の「基本語彙→Unified_Level→PEJVO→PIV」は、**3バンド(基本語彙/PEJVO/PIV)+ 基本語彙バンド内の順序キー=Unified_Level** の意。**3バンド=band_rank 3階層**(rank0: basic/接辞 suf/pref/機能語 prep/correl/num/func, rank1: pejvo/sci/elem/cal/rel, rank2: piv/proper)。実装は `_gas_identifier.ps1` の `$BR` が正典。
- **E は「再考フラグ」止まり** — 最終は意味透明性・日中なじみ優先(一级内なら高画数=藏/壁 も許容)。
- 画数 source = **Unicode Unihan kTotalStrokes**(`通用规范汉字表_一级3500字_画数.tsv`)。
- 重み **a=1.0 / b=0.3 / c=0.1**(自然対数 ln)で確定【確・2026-06-16】。
- **基本形(base)の上書き**【確・2026-06-20】: 同字グループの代表(基本形)は、純P計算だと曜日(月/火…)等が過補正されるため、**2890CSV難易度(Unified_Level)+中心義**で curate した例外表 **`_base_override.tsv`(70群)**で固定する(例 lern=学 / kompren=解 / on=分 / ripet=复 / insekt=虫 を各群の基本形に強制)。CSV難易度は日中エスペランチストが定めた正規化平均値で高信頼。識別子付与(`_gas_identifier.ps1`)はこの語根を群の先頭に強制する。

## 9. 重複漢字と識別子システム ★【確定 2026-06-16(原典GAS = 正典)】

R1で同字を複数語根が共有 → 一意復号のため**優先順位の低い語根に識別子**を付ける(識別子仕様書)。

1. **同じ漢字でグループ化**(1語根なら識別子なし)。
2. グループ内を **P 昇順**(P は §8 の合成値。汎用性 F が支配的に効く)。**P がほぼ同点(差 ≤ 1.0)なら短い語根を上**(識別子仕様書 §3 の同点処理・あなた確定)。
3. **先頭=基本形(識別子なし)**。
4. 2番目以降: **頭文字は必ず保持**。グループ内で**頭文字が唯一なら頭文字のみ**、重なれば**頭文字+差別化文字1字**。
5. 差別化は**子音優先・母音補完**。非基本形の頭文字がグループ内で唯一なら頭文字のみ。頭文字が重複していても、**基本形と異なる頭文字の初出非基本形は頭文字のみ**でよい(例 bulb=球ᴮ, komenc=始ᴷ, minut=分ᴹ — 各群の最初のその頭文字)。同頭文字の2個目以降、または基本形と同頭文字の非基本形は、頭文字後の子音列を**同一頭文字内**の既出位置と照合し初出子音、無理なら母音を加える(例 molekul=分ᴹᴸ[minut=分ᴹ と同頭m], kompleks=复ᴷˢ / komplik=复ᴷᴹ, aŭskult=听ᴬˢ)。**同頭文字内で短い語根が長い語根の語幹になっている場合は短い語根を先に処理し、長い語根は追加末尾子音で差別化**する(例 ambasad=使ᴬ / ambasador=使ᴬᴿ)。多い場合は2文字識別子。**※ 2026-06-16 確定(原典GAS + 仕様書 §0.5)**: ①基本形選択 = 最小band → C昇順(C=band_rank×1000+P)→ **同点(差≤1.0)は短語根**(§3厳密。minut=分ᴹ / molekul=分ᴹᴸ のように頻度高でも長語根は短idを失う)。②差別化 = **位置照合方式**(仕様§5。同じ位置で先行語根が出していない字を子音列→全文字列の順で選ぶ。komplik=复ᴷᴹ / kompleks=复ᴷˢ)。③照合スコープ = **頭文字グループ内(per-head)**。④衝突フォールバック = **別文字**(数字添字でない。子音列→全文字列で最初の未使用字。bio=生ᴮᴼ)。⑤子音で差別化しきれない語根は母音を force-append(例 arkeo=古ᴬᴼ: 同頭『古』群で ar/aĥ が子音を先取済→母音 o)。実装 `_gas_identifier.ps1`(GAS忠実移植・確定例を全再現検証済)。正典 = `エスペラント語根_識別子付与アルゴリズム_仕様書.md §0.5`。
6. 文字種: 母音=a/e/i/o/u のみ。**ŭ は子音**、ĉĝĥĵŝ も子音。
7. ラテン識別子を**上付き字体**(大文字優先。Unicodeに上付き大文字が無い **c/f/s/z は小文字モディファイア**で代替)で漢字直後に表示: `a→ᴬ b→ᴮ c→ᶜ ĉ→ᶜ̂ d→ᴰ e→ᴱ f→ᶠ g→ᴳ ĝ→ᴳ̂ h→ᴴ ĥ→ᴴ̂ i→ᴵ j→ᴶ ĵ→ᴶ̂ k→ᴷ l→ᴸ m→ᴹ n→ᴺ o→ᴼ p→ᴾ r→ᴿ s→ˢ ŝ→ˢ̂ t→ᵀ u→ᵁ ŭ→ᵁ̆ v→ⱽ z→ᶻ`。**※ ĉĝĥĵŝŭ は基底上付き字+結合分音記号(2コードポイント)合成 → 2026-06-16 確定済**(`_make_sidecar.ps1` の `$uni` マップで特殊字未変換0・`⟦…⟧`抽出も両版 diff=0 で問題なし。§15)。
8. **付与は最終の全体パスで一括**(個別段階は最良の意味字を置くだけ)。

**確定例**: `sur=上 / super=上ˢᴾ`(頭s重複→子音p新規)、`krom=外 / ekster=外ᴱ`(頭e唯一)、`ek-=始 / komenc=始ᴷ`。

## 10. 表記正規化(h-system)【確】

- 辞書本体は **h-system**: `c^=ĉ g^=ĝ h^=ĥ j^=ĵ s^=ŝ u^=ŭ`。CSV は Unicode 直書き。
- 内部処理(照合・識別子)は **Unicode 正規形**(ŭ=子音 と u=母音 を区別)。**書き戻しは原本の h-system を維持**。

## 11. 出力フォーマット ★【確】

`見出し(原本のまま)` + **`⟦`** `漢字割当` **`⟧`** + `:` + `定義(原本のまま)`

- `⟦…⟧`(U+27E6/27E7)は辞書・PIV引用・h-system に出現せず `⟦([^⟧]*)⟧:` で一意抽出可。
- 中身は `/` 区切りで**見出しのスラッシュ数と一致**。**語根・接辞→漢字、文法語尾→ラテン字のまま**。
- **連結母音 o は保持(省略しない)**【確・2026-06-20】。`akv/o/fal/o⟦水/o/落/o⟧`・`bel/ec/o/krem/o⟦美/性/o/酪/o⟧` のように **1:1 構造を維持**。理由: 省略すると `美/性酪/o` のように形態素境界を跨いで結合し構造を崩すため。省略が要る場合は後処理に委ねる(`_inject_final.ps1` の `$dropLinkO=$false`)。
- 複数語見出しは `⟦…⟧` 内で半角空白を語境界に。
- **`##` マーカー見出しは注入しない**【確・2026-06-20】。原本の一部見出しに付く `##重複語`(重複エントリ注記)等は、その見出しを注入せず原本のまま出力(重複語は正規見出しで割当済)。`_inject_final.ps1` が `:` より前に `##` を含む見出しをスキップ→繁体字(複/語)の `⟦⟧` 混入を防止。
