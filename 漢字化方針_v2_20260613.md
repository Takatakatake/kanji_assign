## 1. 最終目標と設計思想

作るのは「自然な訳語辞典」ではなく、**規則から未知語の意味を予測できる、閉じた・体系的な表記コード**。対象は日本人・中国人**双方**+**非漢字圏学習者**。簡体字基本。個々の自然さより**透明性・一貫性・最小性**を優先する(これが全判断の上位制約)。

## 2. 用語定義

| 用語 | 定義 |
|---|---|
| 語根 | スラッシュ分解形態素のうち文法語尾以外。接辞も割当対象。 |
| 文法語尾 | -o/-a/-i/-e/-as 等。**漢字化せずラテン字のまま残す**(§3・§12)。 |
| 割当漢字 | 1語根=1漢字、または各字が意味に寄与する透明2字。 |
| 核漢字 | あるカテゴリを代表し複数語根に再利用する字(R1)。 |
| 未対応 | 意味訳不能でラテン語形のまま残す語根。 |
| 識別子 | 同字共有時に優先順位の低い語根へ付す上付き標識(§10)。 |
| 基本形 | 同字グループで P 最小(=最優先)の語根。識別子なし。 |
| P | 優先順位指標。高いほど優先順位が低い(§9)。 |
| h-system | 辞書本体の `c^=ĉ` 等のキャレット表記(§11)。 |

## 3. 割当の対象と単位 ★

- 見出しの**スラッシュ分解を信頼**し、**各語根・接辞に漢字を1対1対応**。勝手な再分解・語尾剥がしはしない(綴り不変・偽分解回避)。【確】
- **文法語尾は漢字化せず、ラテン字のまま `⟦…⟧` 内に残す**。【確・2026-06-13 訂正】
- 接辞・相関詞・前置詞などの機能語も対象。【確】

```
見出し  abel/bred/ist/o
割当    蜂 / 养 / 家 / o      ← 語根→漢字、接辞 ist→家、語尾 o はそのまま
出力    abel/bred/ist/o⟦蜂/养/家/o⟧:養蜂家
```

## 4. 漢字選定の原則・手順

### 4.1 5大原則【確】

| | 原則 | 要点 |
|---|---|---|
| R1 | 漢字種類数の最小化 | 同字・偏旁・熟語を再利用。1字で立つなら1字。2字は各字が意味に寄与する時のみ(冗長2字化不可) |
| R2 | 基礎漢字 | HSK基礎・簡体字・**《通用规范汉字表》一级(3500)**内。専門難字は基礎字の組合せで代替 |
| R3 | 画数・学習効率 | 少画数優先。多画数難字1字より少画数の透明2字 |
| R4 | **意味の透明性(最重要)** | **音訳は部分音訳も含め絶対禁止**。各要素から全体意味が論理的に推測可能(mikroskop→微镜) |
| R5 | 体系的一貫性 | **一貫性 > 自然さ**(不自然でも統一適用。回避は既存語と衝突し誤読を招く時のみ)。接尾辞は完全統一。**同語源・類似音は共通核漢字で系統化**(konkur竞/konkurs竞技)。**多義語根は英語対応語の中心義を基準義**とする(国際的理解の促進)【確】 |

### 4.2 探索順(候補生成)【確】
① HSK基礎1字 → ② 透明な2字熟語 → ③ 部首が意味分野を示し構成から一文で説明できる字 → ④ 未対応。

### 4.3 採否比較順(候補選択)【確】
**① 意味透明性(日中双方) → ② 基礎漢字該当 → ③ 文字数 → ④ 画数。**

### 4.4 日中乖離の裁定(R2c)【確】
主要意味が日中で食い違う字(走・汤・手纸、写=写す・吃=どもる 等)は、**同等に易しい代替があれば回避**。代替が難字しかなければ**中国語側の意味を基準**に採用。
> 実適用例: 好→良 / 房→室 / 写→书 / 吃→食 / 城→市 / 面→颜 / 図→画。

### 4.5 絶対禁止【確】
音訳(部分音訳含む)/ 意味不透明な慣用熟語 / 一级外の専門難字(4.2③を満たす時のみ例外)/ 冗長2字化 / 漢字圏固有名詞への既存表記流用。

## 5. 機能語の確定割当【確】(全90語・サンプル §C)

### 相関詞(45)= 接頭(漢字)+ 語尾(単母音=ラテン / 多字=漢字)
接頭: **ki=何 / ti=那 / i=某 / ĉi=全 / neni=无**。多字語尾: **-am=时 / -al=因 / -om=量 / -es=之 / -el=样**。
例: `kio⟦何o⟧` `kiu⟦何u⟧` `kiam⟦何时⟧` `kial⟦何因⟧` `ĉio⟦全o⟧` `nenie⟦无e⟧`。

### 前置詞(33)
sur上 / super上ˢᴾ / sub下 / antaŭ前 / post后 / apud旁 / inter间 / dum期 / en内 / el出 / **ekster外ᴱ** / **krom外** / kun共 / sen无 / per以 / tra穿 / laŭ按 / por为 / pro因 / pri于 / al向 / ĝis至 / trans越 / kontraŭ对 / de从 / da份 / **je=未対応** / anstataŭ代 / ĉe在 / ĉirkaŭ围 / po每 / preter过 / malgraŭ不顾。

### 数詞(12)
unu一 du二 tri三 kvar四 kvin五 ses六 sep七 ok八 naŭ九 dek十 cent百 mil千。

> **同字共有について**: 機能語間・内容語との同字共有(例 因=-al/pro/kaŭz、无=neni-/sen、共=kun/komun、人=hom/person)は R1 の想定内。相関詞=接頭必須・他=標準形で別キーとなり、最終的に §9 全体識別子パスで一意復号される。

## 7. 固有名詞・未対応【確】

- 意味訳不能(音訳になる)語根 → **未対応 = ラテン語形(h-system)のまま**。漢字/ラテン文字で機械判別可。
- **純粋固有名詞は必ず未対応**。漢字圏の地名・人名も既存漢字表記を流用しない。
- 例: `Pariz/o⟦Pariz/o⟧`、`je`、`-um`。

## 8. 優先順位指標 P ★

「高いほど優先順位が低い」指標。3要素を合成(ユーザー指定):

| 要素 | 定義 | 向き |
|---|---|---|
| **F 汎用性** | 辞書でその語根を含む見出し数 | 高→優先↑ |
| **E 省エネ度** | 語根のラテン文字数 ÷ 割当漢字の総画数 | 高(省エネ)→優先↑ |
| **D 習得難易** | 画数 + 字表ランク | 高(難)→優先↓ |

- 合成 `V = a·log(F+1) + b·E − c·D`。**層(基本語彙→PEJVO→PIV)は別バンドで上書きし跨がせない**。**P = −V**(小=高優先=基本形・良い漢字を優先取得)。
- 層内の基本語彙は **Unified_Level** を第一キー(低いほど優先)。
  - ※ founding prompt の「基本語彙→Unified_Level→PEJVO→PIV」は、**3バンド(基本語彙/PEJVO/PIV)+ 基本語彙バンド内の順序キー=Unified_Level** の意。
- **E は「再考フラグ」止まり** — 最終は意味透明性・日中なじみ優先(一级内なら高画数=藏/壁 も許容)。
- 画数 source = **Unicode Unihan kTotalStrokes**(`通用规范汉字表_一级3500字_画数.tsv`)。
- 重み a/b/c はサンプルで調整(§15)。

## 9. 重複漢字と識別子システム ★【骨子=確定 / 一部=要確認(仕様書6章)】

R1で同字を複数語根が共有 → 一意復号のため**優先順位の低い語根に識別子**を付ける(識別子仕様書)。

1. **同じ漢字でグループ化**(1語根なら識別子なし)。
2. グループ内を **P 昇順**(P は §8 の合成値。汎用性 F が支配的に効く)。**P がほぼ同点(差 ≤ 1.0)なら短い語根を上**(識別子仕様書 §3 の同点処理・あなた確定)。
3. **先頭=基本形(識別子なし)**。
4. 2番目以降: **頭文字は必ず保持**。グループ内で**頭文字が唯一なら頭文字のみ**、重なれば**頭文字+差別化文字1字**。
5. 差別化は**子音優先・母音補完**。非基本形の頭文字がグループ内で唯一なら頭文字のみ。頭文字が重複していても、**基本形と異なる頭文字の初出非基本形は頭文字のみ**でよい(例 molekul=分ᴹ, bulb=球ᴮ, anĝel=使ᴬ)。同頭文字の2個目以降、または基本形と同頭文字の非基本形は、頭文字後の子音列を**同一頭文字内**の既出位置と照合し初出子音、無理なら母音を加える(例 minut=分ᴹᴺ, sobr=节ˢᴮ, aŭskult=听ᴬˢ)。**同頭文字内で短い語根が長い語根の語幹になっている場合は短い語根を先に処理し、長い語根は追加末尾子音で差別化**する(例 ambasad=使ᴬᴹ / ambasador=使ᴬᴿ)。多い場合は2文字識別子。**※ 衝突フォールバック(数字添字 vs 別文字)と「位置照合方式 vs 基本形相違位置方式」の選択は仕様書6章で未収束 → §15・仕様書§6 参照(子音優先の単一方式で回し例外を手当て、を暫定方針)**。
6. 文字種: 母音=a/e/i/o/u のみ。**ŭ は子音**、ĉĝĥĵŝ も子音。
7. ラテン識別子を**上付き字体**(大文字優先。Unicodeに上付き大文字が無い **c/f/s/z は小文字モディファイア**で代替)で漢字直後に表示: `a→ᴬ b→ᴮ c→ᶜ ĉ→ᶜ̂ d→ᴰ e→ᴱ f→ᶠ g→ᴳ ĝ→ᴳ̂ h→ᴴ ĥ→ᴴ̂ i→ᴵ j→ᴶ ĵ→ᴶ̂ k→ᴷ l→ᴸ m→ᴹ n→ᴺ o→ᴼ p→ᴾ r→ᴿ s→ˢ ŝ→ˢ̂ t→ᵀ u→ᵁ ŭ→ᵁ̆ v→ⱽ z→ᶻ`。**※ ĉĝĥĵŝŭ は基底上付き字+結合分音記号(2コードポイント)合成 → フォント依存の表示崩れ・`⟦…⟧`抽出への影響を要実機検証(§15)**。
8. **付与は最終の全体パスで一括**(個別段階は最良の意味字を置くだけ)。

**確定例**: `sur=上 / super=上ˢᴾ`(頭s重複→子音p新規)、`krom=外 / ekster=外ᴱ`(頭e唯一)、`ek-=始 / komenc=始ᴷ`。

## 10. 表記正規化(h-system)【確】

- 辞書本体は **h-system**: `c^=ĉ g^=ĝ h^=ĥ j^=ĵ s^=ŝ u^=ŭ`。CSV は Unicode 直書き。
- 内部処理(照合・識別子)は **Unicode 正規形**(ŭ=子音 と u=母音 を区別)。**書き戻しは原本の h-system を維持**。

## 11. 出力フォーマット ★【確】

`見出し(原本のまま)` + **`⟦`** `漢字割当` **`⟧`** + `:` + `定義(原本のまま)`

- `⟦…⟧`(U+27E6/27E7)は辞書・PIV引用・h-system に出現せず `⟦([^⟧]*)⟧:` で一意抽出可。
- 中身は `/` 区切りで**見出しのスラッシュ数と一致**。**語根・接辞→漢字、文法語尾→ラテン字のまま**。
- 複数語見出しは `⟦…⟧` 内で半角空白を語境界に。
- **原本不可侵**: 作業ファイルから `⟦[^⟧]*⟧` を全除去 → 原本とバイト一致、で漢字注入以外の無改変を証明。

```
abel/o⟦蜂/o⟧:{Ｂ}【虫】ミツバチ
akv/o⟦水/o⟧:{Ｂ}水
bon/a⟦良/a⟧:{Ｂ}良い
mal/grand/a⟦反/大/a⟧:小さい            ［反+大=大の反対］
patr/in/o⟦父/女/o⟧:母                  ［父+女=母。母を新設せず再利用］
skrib/i⟦书/i⟧:書く                     ［skrib=书。书グループの基本形］
libr/o⟦书ᴸ/o⟧:本                       ［libr も书を共有。頭文字 l が唯一→識別子 l→ᴸ］
kio⟦何o⟧:何
Pariz/o⟦Pariz/o⟧:【地名】パリ           ［純粋固有名詞=未対応］
```

> スコア P・サイドカー TSV(語根/漢字/識別子/P/グループキー)は派生生成(機械処理・GAS再処理用)。

## 12. 基礎字の保証(字表照合・字種範囲)【確】

**割当漢字は原則すべて《通用规范汉字表》一级3500内に収める。** これは原典 R2「HSK基礎漢字を最優先・常用漢字範囲」を**機械照合可能に具体化**したもの(基準の差し替えではなく具体化。HSK基礎は依然として選定の最優先)。

**字種の許容ラダー**:
1. 一级の**単字**。
2. 一级字どうしの**透明2字合成**(微+镜=微镜)。
3. (極力回避)探索順③(部首が意味分野を示し構成から一文で説明可)を満たす**一级外字**を、**個別フラグ付き**でのみ。
4. それも無理なら**未対応**。

→ **一级外字が出たら即フラグ**し、③として正当化できるか個別判断、無理なら未対応へ倒す。各バッチで機械照合する(PIV専門語では未対応が増える前提)。

## 15. 残る要確認【要】

- **P の重み a/b/c**: サンプル数十語で実走し調整。
- **P と基本形選択の関係**: E(省エネ度)は長語根を上げる向きで、グループ内基本形の「F同点=短語根」とは逆。→ **基本形選択は F 支配+短語根 tiebreak、E は語彙全体の優先度/価値づけに使う**、という切り分けで良いか確認(暫定はこの切り分け)。
- **識別子の未収束(仕様書6章)**: ①衝突フォールバック(数字 vs 別文字)、②「位置照合(子音優先)」vs「基本形相違位置」方式。暫定=子音優先の単一方式で回し `studado`型の例外だけ手動上書き。
- **識別子の上付き字体の実機検証**: ĉĝĥĵŝŭ の「基底上付き字+結合分音記号」合成が、フォント表示・`⟦…⟧`一意抽出で壊れないか。代替(基底字+独立上付きサーカム / h-system風2文字 / 子音照合で特殊字を回避)も検討。
- **HSK の版**: 一级3500を基礎字の硬基準とする(確定)が、HSK級との対応表も整理するか。
- **多義語根の境界**: 基準義=英語対応中心義(R5確定)。**英語中心義と中国語常用義が衝突する境界ケース**の裁定のみ要確認。
- **固有名詞の意味訳境界**: 語源的地名・元素名・生物分類名をどこまで意味訳するか。
- **milion 以上**: 中国語「万・億」位取りとの整合。
